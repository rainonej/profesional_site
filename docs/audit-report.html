Critical Blockers (must resolve first)

Blocker 1

Dual CMS architecture creates confusion and future breakage risk

The repo currently supports both Pages CMS and Decap CMS assumptions. This affects docs, contributor onboarding, and long-term automation.

  • .pages.yml defines Pages CMS as live path.
  • site/public/admin/* keeps Decap auth/collections as a second path.
  • CI and README focus on GitHub Pages, while Decap config is Netlify Git Gateway-oriented.
Blocker 2

Information architecture and copy still represent a template, not Agreni

Core messaging still says "creative professional," while project brief and planning docs identify the target as equitable learning environments, teacher development, STEM access, public scholarship, and collaboration.

Blocker 3

Booking/contact and settings remain partly hardcoded

Calendly URL still uses PLACEHOLDER in page code, and key strategy fields (booking URL, collaboration CTA, richer socials) are not modeled in editable settings.

Content & Brand Gap Analysis
